## 1. Contrato Oficial Asaas (Evidência P1.6)

Fonte: [Mechanism for validating transfers via webhooks](https://docs.asaas.com/docs/mechanism-for-validating-transfers-via-webhooks)

### Request (Asaas → nosso endpoint)

| Campo | Valor |
|-------|-------|
| Método | `POST` |
| Content-Type | `application/json` |
| Header auth | `asaas-access-token` (token configurado no painel) |
| Timing | ~5 segundos após `POST /v3/transfers` |
| Payload | `{ "type": "TRANSFER", "transfer": { "id", "status", "value", ... } }` |

### Response (nosso endpoint → Asaas)

| Cenário | HTTP | Body |
|---------|------|------|
| Aprovar | `200` | `{ "status": "APPROVED" }` |
| Recusar | `200` | `{ "status": "REFUSED", "refuseReason": "..." }` |

### Comportamento Asaas

| Aspecto | Valor |
|---------|-------|
| Timeout | 10 segundos (Webhooks FAQ) |
| Retry auth | Máx. 3 tentativas; depois cancela transferência |
| Assinatura HMAC | Não — apenas `asaas-access-token` |

## 3. Endpoint

```
POST /webhooks/asaas/transfer-validation
```

Separado de `POST /webhooks/asaas` (webhook financeiro).

Com gate OFF: HTTP `404` + log `[ASAAS][TRANSFER_AUTH] gate_disabled`.

## 4. Variáveis de Ambiente

| Variável | Default | Descrição |
|----------|---------|-----------|
| `ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED` | **OFF** (`false`/ausente) | Gate principal P1.6A |
| `ASAAS_TRANSFER_AUTH_WEBHOOK_TOKEN` | **ausente** | Token `asaas-access-token` |
| `ASAAS_TRANSFER_AUTH_STRICT_MODE` | `true` | Exige token válido |
| `ASAAS_TRANSFER_AUTH_IP_CHECK` | `false` | Allowlist IPs oficiais Asaas |

**Alias legado (P1.6):** `ASAAS_TRANSFER_AUTH_ENABLED`, `ASAAS_TRANSFER_AUTH_TOKEN`.

---

## 5. Critérios de Autorização

Todos devem ser verdadeiros:

- Gate `ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED=true`
- Token válido (se strict mode)
- Payload `type === TRANSFER` com `transfer.id`
- Saque existe (lookup `asaas_transfer_id` ou `payout_external_reference`)
- Provider Asaas (`asaas_transfer_id` / `asaas_payout_raw`)
- Status elegível: `pendente`, `pending`, `aguardando_confirmacao`, `processando`, etc.
- `transfer.status === PENDING`
- Valor ≈ `net_amount` (tolerância R$ 0,01)
- `externalReference` consistente (se presente)
- `correlation_id` presente
- Ledger sem duplicidade / compensação / rollback ativo
- Wallet read-only: usuário existe, saldo finito ≥ 0

---

## 6. Critérios de Negação

| Motivo | Código interno |
|--------|----------------|
| Gate OFF | `GATE_DISABLED` / HTTP 404 |
| Token ausente/inválido | `AUTH_TOKEN_*` |
| Payload incompleto | `TRANSFER_ID_MISSING`, etc. |
| Saque não encontrado | `WITHDRAWAL_NOT_FOUND` |
| Provider não Asaas | `PROVIDER_NOT_ASAAS` |
| Status terminal | `pago`, `cancelado`, `falhou`, `estornado`, etc. |
| Valor divergente | `VALUE_MISMATCH` |
| Ledger inconsistente | `LEDGER_COMPENSATED`, etc. |
| Wallet inconsistente | `WALLET_INCONSISTENT` |
