# F4.3C — Promoção Oficial do Asaas como PSP Primário da Arquitetura V1

**Projeto:** Gol de Ouro™ Backend  
**Data:** 2026-06-27  
**Status:** **PASS**  
**Modo:** arquitetura + documentação — **sem deploy**, **sem commit**, **sem alteração de produção**

---

## 1. Objetivo

Promover oficialmente o **Asaas** como PSP primário da **arquitetura** da V1 (PIX IN + PIX OUT via Payment Engine), preservando Mercado Pago como legado/fallback operacional em produção até homologação explícita.

---

## 2. Distinção obrigatória

| Dimensão | Valor |
|----------|-------|
| **Arquitetura primária** | `PRIMARY_PSP=asaas` (default implícito) |
| **Provider efetivo em produção** | Mercado Pago enquanto `ASAAS_PRODUCTION_ENABLED≠true` |
| **Provider efetivo em dev/sandbox** | Asaas quando `isAsaasProviderResolvable()` (F4.3A ou gate prod) |
| **Legado** | Mercado Pago — fallback quando Asaas não resolvível |
| **Secundário** | Celcoin — `PAYOUT_PROVIDER=celcoin` + flags |

```text
┌─────────────────────────────┐     ┌──────────────────────────────┐
│   Arquitetura Primária      │     │   Provider de Produção       │
│   PRIMARY_PSP = asaas       │  ≠  │   MP legado (gate fechado)   │
│   Factory + contratos       │     │   ASAAS_PRODUCTION_ENABLED   │
└─────────────────────────────┘     └──────────────────────────────┘
```

## 4. Auditoria obrigatória

### 4.1 Todas as chamadas financeiras passam pela Factory?

| Caminho | Passa pela Factory? | Notas |
|---------|:-------------------:|-------|
| Worker payout (`payout-worker.js`) | ✅ | `createPixWithdrawCompat` → `resolvePayoutProvider()` |
| Admin enviar saque (`adminWithdrawController.js`) | ✅ | `createPixWithdrawCompat` (F4.3C) |
| Domínio `processSingleWithdrawalPayout` | ✅ | Usa `createPixWithdrawCompat` internamente |
| `createPixWithdrawCompat.js` | ✅ | Ponte oficial legado → factory |
| **PIX IN criar cobrança** (`server-fly.js`) | ❌ | Inline `services/pix-mercado-pago` / MP API |
| **PIX IN webhook depósito** (`server-fly.js`) | ❌ | Rota `/api/payments/webhook` — MP-specific |
| **PIX OUT webhook** (`server-fly.js`) | ❌ | Rota `/webhooks/mercadopago` — inline |
| `MercadoPagoPayoutProvider` | ✅ | Adapter legado que delega a `pix-mercado-pago.js` |
| Scripts operacionais (`scripts/payouts/*`) | ❌ | Testes/auditoria — import MP direto (não runtime prod) |
| `paymentController.js` / rotas payment | ❌ | Legado MP — fora da factory |

**Conclusão:** PIX OUT via worker/admin **passa pela factory**. PIX IN e webhooks **permanecem no monólito MP** — dívida documentada, fora do escopo de wiring F4.3C.

---

### 4.2 Código novo chamando Mercado Pago diretamente?

Nenhum caminho **novo** introduzido nesta fatia. Ocorrências **pré-existentes** mantidas (legado):

- `server-fly.js` — import `createPixWithdraw`, `getTransactionIntent`
- `services/pix-mercado-pago.js` — serviço legado MP
- `MercadoPagoPayoutProvider.js` — adapter intencional
- Scripts de certificação/auditoria

**Admin e worker** deixaram de importar MP diretamente nesta fatia.

---

### 4.3 Pontos onde Asaas ainda não pode ser resolvido?

| Contexto | Resolvível? | Condição |
|----------|:-----------:|----------|
| Factory PIX OUT (sandbox F4.3A) | ✅ | `ASAAS_TEST_LIVE=1` + guards sandbox |
| Factory PIX OUT (produção) | ⏳ | `ASAAS_PRODUCTION_ENABLED=true` + credenciais prod |
| Factory PIX IN | ✅ sandbox / ⏳ prod | `resolvePaymentProvider()` — null fora de resolvable |
| Monólito PIX IN (`server-fly.js`) | ❌ | Não wired — continua MP |
| Webhooks Asaas | ✅ sandbox | Rotas/engine Asaas; monólito MP separado |
| `authorized=false` transfer sandbox | ⚠️ | Limitação operacional Sandbox Asaas (F4.2G.3) |

### 4.4 Dependências ocultas do Mercado Pago?

| Dependência | Local | Impacto |
|-------------|-------|---------|
| PIX IN produção | `server-fly.js`, `paymentController.js` | **Alto** — fluxo principal depósito |
| Webhooks MP | `/api/payments/webhook`, `/webhooks/mercadopago` | **Alto** |
| Colunas `mp_*` em `saques` | Schema DB | **Médio** |
| `isPixPayoutConfigured()` | `services/pix-mercado-pago.js` | **Baixo** — usado só quando payout efetivo = MP |
| Tokens env | `MERCADOPAGO_*` | **Alto** em prod atual |
| Assinatura webhook Ed25519 | `utils/webhook-signature-validator.js` | **Médio** — MP payout |
