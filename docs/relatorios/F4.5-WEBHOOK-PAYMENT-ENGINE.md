## 3. Arquitetura antes / depois

### Antes

```text
POST /api/payments/webhook → validação MP inline → Supabase idempotência → claimAndCreditApprovedPixDeposit
```

### Depois (com `PAYMENT_WEBHOOK_ENGINE_ENABLED=true`)

```text
POST /api/payments/webhook → assinatura MP (preservada) → processPaymentWebhook()
POST /webhooks/asaas       → flags + token Asaas      → processPaymentWebhook() [dry-run sandbox]
```

Com `PAYMENT_WEBHOOK_ENGINE_ENABLED=false` (default): **comportamento legado 100% preservado**.


---

## 9. Validação Mercado Pago

- `WebhookSignatureValidator.validateMercadoPagoWebhook` **preservado**
- Normalização `approved` → `shouldCreditWallet=true`
- Idempotência Supabase via `checkSupabaseDepositIdempotency`
- Crédito via `claimAndCreditApprovedPixDeposit` (injetado do monólito)

---

## 10. Validação Asaas

- Token header `asaas-access-token` (F4.2C)
- `PAYMENT_RECEIVED` → status `approved`, `shouldCreditWallet=true`
- **Dry-run only** — `creditDecision: dry_run_credit_decision_only`
- Produção bloqueada sem `ASAAS_PRODUCTION_ENABLED=true`

---

## 11. Idempotência

| Provider | Mecanismo |
|----------|-----------|
| Mercado Pago | `pagamentos_pix.status=approved` (Supabase) |
| Asaas sandbox | `PaymentWebhookDryRunStore` in-memory |

Teste: segundo evento idêntico → `idempotent: true`, sem crédito duplicado.
