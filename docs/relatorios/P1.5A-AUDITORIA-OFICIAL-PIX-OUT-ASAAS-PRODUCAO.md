**Projeto:** Gol de Ouro™ V1  
**Data:** 2026-06-28  
**Modo:** READ-ONLY ABSOLUTO  
**Contexto:** Pós-certificação P1.4Z (PIX IN Asaas Produção) · Recomendação A1.0: desacoplamento da Payment Engine **após** encerramento V1

## Veredito

# FAIL

**O fluxo PIX OUT Asaas em produção não está implementado nem homologável no estado atual.**  
A operação de saque em produção permanece **Mercado Pago** (transaction-intents + webhook `/webhooks/mercadopago`).  
O Asaas possui **capability sandbox certificada** (F4.2G/F4.3A), mas **não existe equivalente produção** ao gate PIX IN (`isAsaasPixInProductionHttpEnabled`).

Esta auditoria **não alterou** código, banco, flags, secrets ou deploy.

### O que funciona hoje (MP)

```
POST /api/withdraw/request → débito wallet + ledger saque/taxa + saques.pendente
    ↓
payout-worker (createPixWithdrawCompat*) → **MP** em produção efetiva
    ↓
POST /webhooks/mercadopago → payout_confirmado / rollback
```

\* Worker usa `createPixWithdrawCompat`; em produção com gate fechado, factory resolve **MercadoPagoPayoutProvider**.

### O que existe para Asaas (sandbox only)

```
isAsaasPrimarySandboxMode() === true
    ↓
AsaasProvider.createPixWithdraw → POST /transfers (httpGate: pixOut)
    ↓
Transfer PENDING / authorized:false (limitação sandbox F4.2G.3)
    ↓
Sem fechamento webhook → saques/ledger Supabase
```

### Factory e resolução

| Componente | PIX OUT suportado? | Evidência |
|------------|-------------------|-----------|

| `processPaymentWebhook` | ❌ Sem payout | Só depósitos Asaas |
| Compat layer MP | ✅ | `MercadoPagoPayoutProvider` |

### Lacunas da Payment Engine para PIX OUT Asaas integral

1. **Sem ramo produção** no provider Asaas (stub terminal).  
2. **Webhook engine** não processa transferências para fechar saque.  
3. **`processPendingWithdrawals`** assume resposta MP (`mp_transaction_intent_id`, `MP_TERMINAL_FAIL` — L1400–1448).  
4. **`server-fly.js` e admin** bypassam factory (MP hardcoded).  
5. **Sem gate PIX OUT produção** simétrico ao P1.3F/P1.4 PIX IN.

**Resposta:** A Payment Engine **não suporta integralmente** PIX OUT Asaas produção. Suporta abstração de interface e sandbox; produção efetiva = MP.

## 15. Conclusão

A auditoria P1.5A mapeou integralmente o fluxo PIX OUT e concluiu:

1. **Sandbox Asaas PIX OUT** — capability validada (F4.2G/F4.3A).  
2. **Produção Asaas PIX OUT** — **não implementada**; homologação **impossível** no estado atual.  
3. **Produção operacional de saques** — **Mercado Pago**, com wallet/ledger/idempotência maduros no request e confirmação MP.  
4. **Payment Engine** — abstração parcial; lacunas críticas em provider prod, webhook transfer e persistência.
