## Causa raiz (P1.4)

A RPC Supabase `claim_and_credit_approved_pix_deposit` valida `p_mercadopago_id` com regex `^\d+$`. IDs Asaas (`pay_1zr8bxbd5abypkws`) retornam:

```json
{ "ok": false, "error": "invalid_mercadopago_id", "credited": false }
```

O handler em `server-fly.js` tratava `ok: false` como falha terminal **sem acionar** o fallback JS existente.

---

## Correção implementada

### Novo módulo

`src/finance/deposit/claimApprovedPixDeposit.js`

| Função | Papel |
|--------|-------|
| `isAsaasProviderPaymentId` | Detecta IDs `pay_[a-z0-9]+` |
| `shouldUseJsFallbackForRpcRejection` | Aciona fallback só quando RPC retorna `invalid_mercadopago_id` + ID Asaas |
| `claimApprovedPixDepositJsFallback` | Claim `pagamentos_pix` pending→approved + ledger + saldo (idempotente via `correlation_id`) |
| `claimApprovedPixDeposit` | Orquestra RPC MP → fallback Asaas |

**Fluxo fallback (idempotente):**
1. Verificar `ledger_financeiro` por `correlation_id=payment_id` + `tipo=deposito` → retorno idempotente se existir
2. Claim atômico em `pagamentos_pix` (`payment_id` ou `external_id`, `status != approved`)
3. `createLedgerEntry` tipo `deposito`
4. Incremento único em `usuarios.saldo`

### Integração

`server-fly.js` — `claimAndCreditApprovedPixDeposit` delega ao módulo P1.4F.
