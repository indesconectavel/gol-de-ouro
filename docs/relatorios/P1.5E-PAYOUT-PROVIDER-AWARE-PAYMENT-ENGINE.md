```text
Worker → processSingleWithdrawalPayout
              ↓
         createPixWithdrawCompat (já wired)
              ↓
         Provider (factory)
              ↓
         Persistência SEMPRE mp_*  ← gap P1.5E
```

- Factory e compat layer já existiam desde F4.3C.

## 5. Fluxo Atualizado

### Pontos de execução mapeados

| Componente | Papel |
|------------|-------|
| `src/workers/payout-worker.js` | Ciclo worker → `processPendingWithdrawals` + `createPixWithdrawCompat` |
| `src/domain/payout/processPendingWithdrawals.js` | `processSingleWithdrawalPayout` — lock, ref, payout, persistência |
| `src/finance/compat/createPixWithdrawCompat.js` | Ponte legado → `requestPixPayout` |
| `src/finance/factory/FinanceProviderFactory.js` | Resolução MP / Asaas / Celcoin / Mock |
| `MercadoPagoPayoutProvider` | Delega `services/pix-mercado-pago.createPixWithdraw` |
| `AsaasPayoutProvider` | Delega `AsaasProvider.createPixWithdraw` |
| `server-fly.js` | Worker cron + admin — **ainda injeta MP direto** (validação only; execução usa compat) |

### Decisão P1.5E

`processSingleWithdrawalPayout` **sempre executa** via `createPixWithdrawCompat` (Payment Engine). Parâmetro `createPixWithdraw` injetado permanece para validação de presença — unificação total em **P1.5F**.

## 7. Integração Asaas

| Modo | Comportamento |
|------|---------------|
| Gates OFF + `PAYOUT_PROVIDER=asaas` | `guardAsaasPixOutProduction()` → `ASAAS_PIX_OUT_PRODUCTION_DISABLED`; sem HTTP |
| Gates ON (mock test) | Factory → Asaas; `POST /transfers` mockado; persistência `asaas_*` |

Status Asaas não-terminal (`PENDING`, `BANK_PROCESSING`) → `aguardando_confirmacao` (paridade MP).

Status terminal Asaas (`FAILED`, `CANCELLED`) → rollback + ledger `falha_payout`.

---

## 8. Persistência Provider-Aware

Módulo `payoutProviderPersistence.js`:

| Provider | Colunas gravadas |
|----------|------------------|
| `mercadopago` | `mp_transaction_intent_id`, `mp_payout_status`, `mp_payout_raw`, `last_mp_sync_at` |
| `asaas` | `asaas_transfer_id`, `asaas_transfer_status`, `asaas_payout_raw`, `last_asaas_sync_at` |

Regra: **nunca** gravar colunas do provider oposto na mesma operação.
