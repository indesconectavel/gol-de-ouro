### Endpoints oficiais identificados

| Operação | Método | Path | Uso |
|----------|--------|------|-----|
| Criar transferência Pix/TED | `POST` | `/v3/transfers` | PIX OUT para chave Pix ou conta bancária |
| Consultar transferência | `GET` | `/v3/transfers/{id}` | Status pós-criação |
| Saldo disponível | `GET` | `/v3/finance/balance` | Pré-requisito: saldo na conta Asaas |

### Autenticação

| Item | Valor |
|------|-------|
| Header | `access_token` |
| Sandbox URL (oficial) | `https://api-sandbox.asaas.com/v3` |
| Sandbox URL (projeto) | `https://sandbox.asaas.com/api/v3` (`.env` local — compatível F4.2A–F4.2E) |

### Parâmetros PIX OUT (chave Pix)

| Campo | Obrigatório | Descrição |
|-------|-------------|-----------|
| `value` | Sim | Valor da transferência |
| `pixAddressKey` | Sim* | Chave Pix destino |
| `pixAddressKeyType` | Sim* | `CPF`, `CNPJ`, `EMAIL`, `PHONE`, `EVP` |
| `operationType` | Não | `PIX` ou `TED` (auto se omitido) |
| `description` | Não | Descrição (Pix) |
| `externalReference` | Não | ID no sistema integrador |

\* Alternativa: objeto `bankAccount` para TED/conta bancária.

1. Conta Sandbox com **saldo disponível** ([Adding balance to Sandbox account](https://docs.asaas.com/docs/adding-balance-to-a-sandbox-account))
2. API Key Sandbox (`$aact_hmlg_...`)
3. Para homologação Pix: chaves fictícias BACEN **ou** chaves de outras contas Sandbox

## 4. Etapa 2 — Capability Check

### O Sandbox suporta PIX OUT?

**SIM**

**Justificativa:**

1. Documentação oficial descreve transferências Pix no Sandbox com endpoints `POST /v3/transfers` e guia dedicado [Testing Transfers](https://docs.asaas.com/docs/testing-transfers).
2. Ambiente Sandbox permite simulação completa de débito (e crédito entre contas Sandbox reais).
3. Implementação mínima `createSandboxPixTransfer()` preparada e testável via script local.

**Ressalvas:** operação depende de saldo Sandbox e possíveis gates de conta (autorização SMS). Isso caracteriza **SUPPORTED_WITH_LIMITATIONS** na prática operacional, não ausência de suporte.

---

## 5. Etapa 3 — Implementação mínima

### Flags

| Flag | Default | Efeito |
|------|---------|--------|
| `ASAAS_PIX_OUT_ENABLED` | `false` | Gate PIX OUT |
| `ALLOW_ASAAS_SANDBOX_PIX_OUT` | `0` | Permite HTTP PIX OUT sandbox |

Gate HTTP: `ASAAS_ENABLED=true` + `ALLOW_ASAAS_SANDBOX_AUTH=1` + `ASAAS_ENV=sandbox` + flags PIX OUT.

### Método de capability

`AsaasPayoutProvider.createSandboxPixTransfer()`:

- Valida flags e ambiente Sandbox
- `GET /finance/balance` (opcional)
- `POST /transfers` com chave BACEN fictícia padrão
- `GET /transfers/{id}` se transferência criada
- Retorna `financialEffect: false`, `capabilityOnly: true`, `integratedInGolDeOuro: false`

### O que **não** foi alterado

- `requestPixPayout()` → continua stub (`ASAAS_PAYOUT_NOT_IMPLEMENTED`)
- `PAYOUT_PROVIDER=asaas` → factory continua bloqueando
- Wallet, ledger, gameplay, MP, Celcoin, banco, rotas públicas
