# F4.4 — PIX IN via Payment Engine com Asaas

**Projeto:** Gol de Ouro™ Backend  
**Data:** 2026-06-27

**Modo:** arquitetura + sandbox — **sem deploy**, **sem commit**, **sem webhook**, **sem wallet/ledger**

## 1. Objetivo

Migrar a **criação de cobrança PIX IN** para a Payment Engine, com Asaas como provider primário arquitetural (F4.3C) e Mercado Pago como fallback legado, sem alterar produção automaticamente.

---

## 3. Estratégia de compatibilidade (3 camadas)

```text
Camada 3 — Monólito
  server-fly.js → createPixDepositCompat()

Camada 2 — Payment Engine
  createPixDeposit() → FinanceProviderFactory.resolvePaymentProvider()

Camada 1 — Provider
  AsaasPaymentProvider.createPixDepositCharge()
  MercadoPagoPaymentProvider.createPixDeposit() [fallback]
```

## 8. Comportamento

### Sandbox controlado (F4.4 profile)

- Factory resolve `AsaasPaymentProvider`
- `createPixDeposit` cria cobrança via API Asaas
- MP **não** é invocado (guard nos testes)

### Produção (gate fechado)

- Factory resolve `MercadoPagoPaymentProvider`
- Fluxo legado preservado via adapter
- Asaas **não** ativa automaticamente

### Fallback legado

- Quando `PAYMENT_PROVIDER=asaas` efetivo mas Asaas não resolvível → MP adapter
- `paymentLegacyFallbackActive=true` no health snapshot
