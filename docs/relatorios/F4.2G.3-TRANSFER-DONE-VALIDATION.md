## 6. Estado terminal (Etapa 3)

| Classificação | Valor |
|---------------|-------|
| **Estado terminal alcançado** | **Não** |
| **Status final** | `PENDING` |
| **`authorized` final** | `false` |
| **Classificação F4.2G.3** | `VALIDATED_WITH_OPERATIONAL_LIMITATION` |

### Motivo documentado

1. A conta Sandbox possui **autorização crítica habilitada** (padrão Asaas).
2. O token `000000` enviado no body (`authToken`) **não foi reconhecido** como autorização válida pela API — campo `authorized` permaneceu `false`.
3. A OpenAPI de `POST /transfers` **não documenta** o parâmetro `authToken`; a forma exata de envio do token crítico não está especificada na referência técnica.
4. A recipe oficial indica que chaves BACEN fictícias devem ir a `DONE` imediatamente — porém isso pressupõe transferência **já autorizada**; com `authorized: false`, o fluxo fica bloqueado em `PENDING`.
5. Autorização manual via **painel Sandbox** permanece como único caminho oficial alternativo não testado automaticamente nesta fatia.

**Conclusão:** a não evolução para `DONE` é **limitação operacional do Sandbox** (autorização crítica + ambiguidade do token na API), **não** falha da integração implementada (criação, consulta e saldo funcionam corretamente).
