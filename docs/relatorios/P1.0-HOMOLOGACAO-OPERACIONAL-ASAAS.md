## 3. Etapa 1 — Auditoria `POST /webhooks/asaas`

### 3.1 Endpoint

| Item | Valor |
|------|-------|
| Rota | `POST /webhooks/asaas` |
| Arquivo | `server-fly.js` (L3550+) |
| Compat layer | `processPaymentWebhookCompat({ provider: 'asaas' })` |
| Engine | `processPaymentWebhook.js` (F4.5) |

### 3.2 Fluxo

```text
POST /webhooks/asaas
  → isPaymentWebhookEngineEnabled()?  → 404 se false
  → isAsaasWebhookRouteAllowed()?     → 403 se bloqueado
  → processPaymentWebhookCompat
       → validateAsaasWebhook (header asaas-access-token)
       → normalizeAsaasPaymentWebhook
       → [sandbox] dry-run / controlled credit (F4.6)
       → [produção + gate ON] Supabase idempotência + claim (P1.0)
  → HTTP 200 { received, engine, provider, ... }
```

### 3.3 Injeção de provider/deps

| Dependência | Antes P1.0 | Após P1.0 |
|-------------|------------|-----------|
| `financeLog` | ✅ | ✅ |
| `supabase` | ❌ | ✅ |
| `claimAndCreditApprovedPixDeposit` | ❌ | ✅ |
| `controlledCreditStore` | ❌ (sandbox F4.6 via scripts) | ❌ (intencional — sandbox only) |

### 3.4 Compatibilidade

| Aspecto | Status |
|---------|--------|
| Contrato evento interno (F4.5) | ✅ preservado |
| Validador token Asaas (`asaas-access-token`) | ✅ |
| Normalizer `PAYMENT_RECEIVED` → `shouldCreditWallet` | ✅ |
| MP legado `/api/payments/webhook` | ✅ inalterado |
| MP payout `/webhooks/mercadopago` | ✅ inalterado |

### 3.5 Pontos de integração

| Ponto | Arquivo | Função |
|-------|---------|--------|
| Rota HTTP | `server-fly.js` | `app.post('/webhooks/asaas')` |
| Compat | `processPaymentWebhookCompat.js` | bridge Express ↔ engine |
| Engine | `processPaymentWebhook.js` | ramo `provider === 'asaas'` |
| Validador | `asaas-webhook-validator.js` | token + classificação evento |
| Normalizer | `asaas-webhook-normalizer.js` | evento padrão interno |
| Idempotência prod | `paymentWebhookIdempotency.js` | `checkSupabaseDepositIdempotency` |
| Crédito prod | `server-fly.js` | `claimAndCreditApprovedPixDeposit` → RPC Supabase |
| Gates | `primary-psp.js`, `asaas-config.js`, `payment-webhook-config.js` | flags em camadas |

### 3.6 Achado pré-implementação

**FAIL inicial:** com `ASAAS_PRODUCTION_ENABLED=true`, o engine retornava `403 blocked_prod` (via `getControlledCreditBlockReason`) e o monólito não injetava handler de crédito real.

**Resolução P1.0:** ramo `isAsaasProductionWebhookCreditEnabled()` + deps Supabase no monólito.

### 3.7 Veredicto Etapa 1

**PASS**

## 4. Etapa 2 — Wire definitivo produção

### 4.1 Condição de ativação

```text
PAYMENT_WEBHOOK_ENGINE_ENABLED=true
AND ASAAS_PRODUCTION_ENABLED=true
AND NODE_ENV=production
AND ASAAS_WEBHOOK_ENABLED=true
```

### 4.2 Cadeia quando gate aberto

```text
Webhook oficial Asaas (POST /webhooks/asaas)
        ↓
Payment Engine (processPaymentWebhook)
        ↓
validateAsaasWebhook + normalizeAsaasPaymentWebhook
        ↓
checkSupabaseDepositIdempotency (pagamentos_pix approved)
        ↓
claimAndCreditApprovedPixDeposit (RPC claim_and_credit_approved_pix_deposit)
        ↓
Wallet real (usuarios.saldo) + Ledger (via RPC R3 quando disponível)
        ↓
Idempotência (replay → idempotent, sem double credit)
```

### 4.3 Arquivos alterados

| Arquivo | Alteração |
|---------|-----------|
| `src/finance/config/payment-webhook-config.js` | `isAsaasProductionWebhookCreditEnabled()` |
| `src/finance/webhooks/processPaymentWebhook.js` | ramo crédito produção Asaas |
| `src/finance/compat/processPaymentWebhookCompat.js` | resposta `productionCredit` |
| `server-fly.js` | injeção `supabase` + `claimAndCreditApprovedPixDeposit` |
| `.env.example` | documentação gate P1.0 |
| `scripts/verify-asaas-production-webhook-wire.mjs` | **novo** — verificação wire |

### 4.4 Proteções mantidas

- Sandbox continua em dry-run (F4.5) ou controlled credit (F4.6)
- Produção **sem** gate → rota 403 (`asaas_webhook_blocked`)
- Controlled credit **bloqueado** em `NODE_ENV=production`
- Gate default `ASAAS_PRODUCTION_ENABLED=false`
