# F4.3A — Asaas Primary PSP Sandbox Mode — Gol de Ouro V1

**Data:** 2026-06-27  
**Fatia:** integração E2E controlada Asaas como provider principal — **Sandbox/local apenas**  
**Produção:** inalterada — default permanece Mercado Pago

---

## 1. Objetivo

Preparar e validar o Asaas como **provider principal** da V1 em ambiente Sandbox controlado, cobrindo:

- PIX IN → confirmação → webhook → wallet → ledger
- Solicitação de saque → aprovação → PIX OUT → consulta → webhook → status

**Sem** usar Mercado Pago como caminho principal nesta validação.

---

## 2. Flags obrigatórias (Sandbox/local)

```env
PAYMENT_PROVIDER=asaas
PAYOUT_PROVIDER=asaas
ASAAS_ENABLED=true
ASAAS_ENV=sandbox
ASAAS_TEST_LIVE=1
ASAAS_PIX_IN_ENABLED=true
ASAAS_PIX_OUT_ENABLED=true
ALLOW_ASAAS_SANDBOX_AUTH=1
ALLOW_ASAAS_SANDBOX_PIX_IN=1
ALLOW_ASAAS_SANDBOX_PIX_OUT=1
ASAAS_WEBHOOK_ENABLED=true
```

Profile aplicado via `applyAsaasPrimarySandboxProfile()` em `scripts/helpers/asaas-test-env.mjs`.

**Proteção produção:** `isAsaasPrimarySandboxMode()` exige `NODE_ENV≠production`, `ASAAS_ENV=sandbox`, `ASAAS_TEST_LIVE=1` e todos os guards — factory **rejeita** `PAYMENT_PROVIDER=asaas` / `PAYOUT_PROVIDER=asaas` fora desse modo.

## 9. Limitações

1. **Ledger in-memory** — E2E não persiste em banco real (proteção produção).
2. **Transferência `PENDING`** — autorização crítica Sandbox (documentado F4.2G.2/G.3).
3. **Webhook handler** — valida eventos; `financialEffect: false` no handler (crédito aplicado pelo orchestrator E2E conforme regra V1).
4. **Rotas HTTP públicas** — não expostas; webhooks simulados localmente.
5. **Produção Fly/Vercel** — inalterada; default factory continua Mercado Pago sem flags F4.3A.
