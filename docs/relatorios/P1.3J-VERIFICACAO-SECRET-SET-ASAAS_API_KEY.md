## 3. Interpretação

| Hipótese | Plausível? |
|----------|:----------:|
| Chave produção gravada com sucesso | **Não** — `length=0`, digest inalterado |
| `secrets set` gravou string vazia de novo | **Sim** — padrão idêntico a P1.3H/P1.3I |
| PowerShell expandiu/stripou o valor no `flyctl` | **Muito provável** |
| Sobrescrita em runtime por código | **Não** — `/proc/$PID/environ` confirma vazio no kernel |

### 3.1 Causa provável: quoting no PowerShell

No Windows PowerShell, construções como:

```powershell
flyctl secrets set ASAAS_API_KEY="CHAVE_PRODUCAO" -a goldeouro-backend-v2
```

podem resultar em valor vazio quando:

- a variável entre aspas não existe ou expande para vazio;
- aspas duplas são interpretadas pelo shell antes do `flyctl` receber o argumento;
- a chave real não foi colada literalmente (placeholder no comando).

**Evidência:** digest inalterado + `length=0` após release = **mesmo conteúdo vazio** de antes.

---

## 4. Comando recomendado (operador)

Colar a chave **literalmente** com aspas simples (evita expansão PowerShell):

```powershell
fly secrets set ASAAS_API_KEY='$aact_prod_COLE_A_CHAVE_COMPLETA_AQUI' -a goldeouro-backend-v2
```

**Não usar** variável de ambiente local nem aspas duplas com expansão.

### 4.1 Validação pós re-set (read-only)

Após novo `secrets set`, confirmar **sem expor valor**:

| Critério | Esperado se OK |
|----------|----------------|
| Digest Fly `ASAAS_API_KEY` | **Diferente** de `49bb8a67b557e235` |
| Runtime `length` | **> 0** |
| `isProdPrefix` | **true** (`$aact_prod_`) |
| `sha256Prefix` | **não null** |

## 5. Impacto operacional

Com `ASAAS_API_KEY` vazia, permanecem bloqueados:

- PIX IN real Asaas produção (P1.3R)
- Chamadas HTTP `api.asaas.com`

Flags atuais (`ASAAS_ENABLED=true`, `ASAAS_ENV=production`, `ASAAS_PIX_IN_ENABLED=true`, `ASAAS_PRODUCTION_ENABLED=false`) **não** compensam chave ausente.
