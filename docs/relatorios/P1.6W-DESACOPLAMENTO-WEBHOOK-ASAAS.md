# P1.6W — Desacoplamento do Webhook Asaas dos Gates PIX OUT

**Data:** 2026-06-30  
**Modo:** CORREÇÃO ESTRUTURAL CONTROLADA

## Veredito

# PASS

O webhook financeiro `POST /webhooks/asaas` **não responde mais HTTP 403** quando `ASAAS_PRODUCTION_ENABLED=false` e `ASAAS_WEBHOOK_ENABLED=true`. PIX OUT permanece protegido pelos gates próprios. Webhook de autorização permanece isolado em `ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED`. Nenhuma operação PIX foi executada.

---

# 1. Resumo executivo

| Problema | Correção |
|----------|----------|
| `ASAAS_PRODUCTION_ENABLED` bloqueava ingresso do webhook | `isAsaasWebhookRouteAllowed()` usa apenas **`ASAAS_WEBHOOK_ENABLED`** em produção |
| HTTP 403 penalizava fila Asaas | Eventos com gate financeiro fechado retornam **HTTP 200** + `ignored` + log |
| Acoplamento monolítico produção | Separação: **ingresso webhook** ≠ **crédito financeiro** ≠ **PIX OUT** |

**Estado operacional recomendado (produção estável):**

| Secret | Função |
|--------|--------|
| `ASAAS_WEBHOOK_ENABLED=true` | Aceitar eventos Asaas (sempre ON em prod) |
| `ASAAS_PRODUCTION_ENABLED=true` | Crédito PIX IN / cobranças em produção |
| `ASAAS_PIX_OUT_PRODUCTION_ENABLED` | Somente `POST /v3/transfers` / payout |
| `ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED` | Somente `/transfer-validation` |

---

# 2. Arquitetura anterior

```text
POST /webhooks/asaas
  → PAYMENT_WEBHOOK_ENGINE_ENABLED?  → 404 se OFF
  → isAsaasWebhookRouteAllowed()
       → ASAAS_WEBHOOK_ENABLED
       → ASAAS_PRODUCTION_ENABLED  ← MONOLÍTICO (P1.6D)
  → 403 asaas_webhook_blocked
  → processPaymentWebhook
       → blocked_prod → 403 (penalização Asaas)
```

**Efeito colateral:** homologações P1.6F fechavam `ASAAS_PRODUCTION_ENABLED=false` → Asaas recebia **403** → penalização de fila → perda de eventos `PAYMENT_RECEIVED`.

---

# 3. Arquitetura nova

```text
POST /webhooks/asaas
  → PAYMENT_WEBHOOK_ENGINE_ENABLED?  → 404 se OFF
  → isAsaasWebhookRouteAllowed()
       → produção: ASAAS_WEBHOOK_ENABLED apenas
       → sandbox: ASAAS_WEBHOOK_ENABLED + ALLOW_ASAAS_SANDBOX_WEBHOOK
  → 403 somente se ASAAS_WEBHOOK_ENABLED=false
  → processPaymentWebhook
       → validação token / normalização
       → PAYMENT_*: crédito se isAsaasProductionWebhookCreditEnabled()
            (ainda exige ASAAS_PRODUCTION_ENABLED)
       → sem crédito: HTTP 200 ignored + deposit_webhook_event_ignored
       → TRANSFER_*: processamento ou ignored (sem 403 por gate PIX OUT)

POST /webhooks/asaas/transfer-validation
  → ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED apenas (inalterado P1.6A)

POST /v3/transfers / approve-and-send
  → ASAAS_PIX_OUT_PRODUCTION_ENABLED + gates payout (inalterado)
```

| `ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED` | `/transfer-validation` | `asaas-transfer-auth-config.js` |
| `PAYMENT_WEBHOOK_ENGINE_ENABLED` | Engine webhook global | `payment-webhook-config.js`, `server-fly.js` |

## 4.2 Retornos HTTP — `/webhooks/asaas`

| Condição | Antes | Depois |
|----------|-------|--------|
| `PAYMENT_WEBHOOK_ENGINE_ENABLED=false` | 404 | 404 (inalterado) |
| `ASAAS_WEBHOOK_ENABLED=false` | 403 | 403 (inalterado) |
| `ASAAS_WEBHOOK_ENABLED=true` + `ASAAS_PRODUCTION_ENABLED=false` | **403** | **200** (ignored ou processado) |
| Token inválido | 401 | 401 (inalterado) |
| `PAYMENT_RECEIVED` + gate crédito ON | 200 + crédito | 200 + crédito (inalterado) |

# 6. Diff resumido

### `isAsaasWebhookRouteAllowed()` — antes

```javascript
if (!isAsaasSandboxWebhookAllowed() && !isAsaasProductionEnabled()) return false;
if (isProductionRuntime() && !isAsaasProductionEnabled()) return false;
```

### `isAsaasWebhookRouteAllowed()` — depois

```javascript
if (isProductionRuntime()) return true; // exige ASAAS_WEBHOOK_ENABLED acima
return isAsaasSandboxWebhookAllowed();
```

### `blocked_prod` — antes → depois

```javascript
// antes: rejected: true, httpStatus: 403
// depois: success: true, ignored: true, creditDecision: 'production_credit_gate_closed'
financeLog('deposit_webhook_event_ignored', { reason: 'production_credit_gate_closed', ... });
```

# 7. Impactos

| Área | Impacto |
|------|---------|
| Fila webhooks Asaas | **Positivo** — 403 por gate fechado eliminado; conformidade doc Asaas (somente 200 = sucesso) |
| PIX IN com `ASAAS_PRODUCTION_ENABLED=true` | **Neutro** — crédito inalterado |
| PIX IN com gate financeiro OFF | **Mudança** — HTTP 200 ignored (antes 403); sem crédito wallet |
| PIX OUT | **Neutro** — gates `ASAAS_PIX_OUT_PRODUCTION_ENABLED` inalterados |
| Auth webhook | **Neutro** — gate próprio |
| Homolog scripts `closeGates()` | Podem manter `ASAAS_PRODUCTION_ENABLED=false` sem penalizar webhook se `ASAAS_WEBHOOK_ENABLED=true` |
