# F4.3B — Asaas E2E com Wallet/Ledger Real em Ambiente Controlado

**Data:** 2026-06-27  
**Fatia:** compatibilidade Wallet/Ledger schema V1 — **sem Supabase produção**  
**Pré-requisito:** F4.3A PASS

---

## 1. Objetivo

Comprovar que o fluxo Asaas E2E é compatível com a **estrutura real** de persistência financeira do Gol de Ouro (`usuarios`, `pagamentos_pix`, `saques`, `ledger_financeiro`), em ambiente **totalmente isolado** da produção.

---

## 2. Decisão de ambiente (auditoria prévia)

| Opção | Descrição | Avaliação |
|-------|-----------|-----------|
| **A** — Supabase Sandbox/Clone | `uatszaqzdqcwnfbipoxg` ou `STAGING_SUPABASE_*` | ⚠️ `STAGING_NOT_CONFIGURED` (F2.2B.2); credenciais ausentes neste workspace; schema legado com drift |
| **B** — Dry-run persistente local | JSON local espelhando schema real | ✅ **Escolhida** — zero risco prod; schema V1 fiel |
| **C** — Transação + rollback em DB | Requer conexão PG | ❌ Rejeitada — risco se `.env` apontar prod (`gayopagjdrkcmkirmfvy`) |

### Opção escolhida

```text
B — Modo Dry-Run Persistente (schema real, persistência local temporária)
```

### Justificativa

1. `.env` local contém referências a **produção** `gayopagjdrkcmkirmfvy` (F3.3A, F6.1A).
2. `STAGING_SUPABASE_URL` **não configurado** (F2.2B.2).
3. Staging `uatszaqzdqcwnfbipoxg` existe documentalmente mas **não é confiável** neste workspace (ENOTFOUND, drift PascalCase).
4. Opção B replica tabelas/colunas V1, constraints de idempotência ledger e persistência em `scripts/.controlled-db/` (gitignored).
5. Guardrail: se ref prod detectada → força Opção B mesmo com `ASAAS_E2E_USE_SUPABASE=1`.

**Recomendação futura:** criar Supabase Sandbox dedicado + `CONTROLLED_SUPABASE_URL` para habilitar Opção A com `ASAAS_E2E_USE_SUPABASE=1`.

## 10. Limitações

1. **Opção B** — persistência local, não Supabase real (proteção intencional).
2. Transferência **`PENDING`** — autorização crítica Sandbox (F4.2G.3 / F4.3A).
3. RPC `claim_and_credit_approved_pix_deposit` **não invocada** — exige ID numérico MP; crédito Asaas usa adapter controlado equivalente V1.
4. Opção A disponível futuramente com Supabase Sandbox dedicado.
