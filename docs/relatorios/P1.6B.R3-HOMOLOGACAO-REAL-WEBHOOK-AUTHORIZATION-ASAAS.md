## 7. Preparação necessária (antes de retomar R3)

### 7.1 Criar saque técnico elegível

1. Autenticar na conta técnica (`validacao.pontaaponta.*` ou `vale2e*`)
2. Confirmar chave PIX cadastrada (CPF/CNPJ se necessário)
3. Solicitar saque via fluxo oficial do app/API:
   - Valor mínimo: **R$ 10** (líquido ≥ R$ 8 após taxa)
   - Status resultante: `pendente`
4. Registrar `saque_id`, `correlation_id`, wallet antes

### 7.2 Confirmar painel Asaas

| Campo | Valor |
|-------|-------|
| URL | `https://goldeouro-backend-v2.fly.dev/webhooks/asaas/transfer-validation` |
| Token | Mesmo de `ASAAS_TRANSFER_AUTH_WEBHOOK_TOKEN` no Fly |
| Mecanismo | Habilitado |
| Autorização crítica (SMS) | Solicitar desativação ao account manager |

### 7.3 Retomar homologação

```bash
# 1. Auditoria
node scripts/p16b-r3-audit-saques.mjs
# Esperado: READY + recommendedSaqueId

# 2. Abrir janela (uma tentativa)
fly secrets set \
  ASAAS_PRODUCTION_ENABLED=true \
  ASAAS_PIX_OUT_PRODUCTION_ENABLED=true \
  PAYMENT_ENGINE_PIXOUT_ENABLED=true \
  PAYOUT_PIX_ENABLED=true \
  PAYOUT_PROVIDER=asaas \
  -a goldeouro-backend-v2

# 3. Executar UMA vez (fly ssh ou script homolog)
P15Y_PHASE=execute P15Y_ALLOW_REAL_PAYOUT=1 P15Y_SAQUE_ID=<id> ...

# 4. Capturar logs
fly logs -a goldeouro-backend-v2 | findstr TRANSFER_AUTH

# 5. Fechar gates (obrigatório)
fly secrets set \
  ASAAS_PRODUCTION_ENABLED=false \
  ASAAS_PIX_OUT_PRODUCTION_ENABLED=false \
  PAYMENT_ENGINE_PIXOUT_ENABLED=false \
  PAYOUT_PIX_ENABLED=false \
  -a goldeouro-backend-v2
fly secrets unset PAYOUT_PROVIDER -a goldeouro-backend-v2
```
