## 4. Comportamento da factory

| Cenário | Comportamento |
|---------|---------------|
| Default (sem `PAYOUT_PROVIDER`) | **Mercado Pago** como payout ativo |
| `PAYOUT_PROVIDER=asaas` | **Erro no boot** — não wired (F4.2B stub) |
| `ASAAS_ENABLED=false` | Asaas aparece apenas em `getHealthSnapshot()` como registrado/desligado |
| `getHealthSnapshot()` | Inclui `asaasEnabled`, `asaasHttpEnabled`, `asaasConfigured`, `registeredPayoutProviders` |

Mercado Pago **permanece default**. Asaas **não substitui** nenhum provider em produção.

## 5. Matriz de flags

| Flag | Default | Efeito |
|------|---------|--------|
| `ASAAS_ENABLED` | `false` | Gate geral Asaas |
| `ALLOW_ASAAS_SANDBOX_AUTH` | `0` | Gate HTTP sandbox |
| `ASAAS_ENV` | `sandbox` | Ambiente (HTTP exige `sandbox`) |
| `ASAAS_API_KEY` | (vazio) | Credencial API Key |
| `ASAAS_BASE_URL` | `https://sandbox.asaas.com/api/v3` | Base URL |

**Novas flags:** nenhuma — reutiliza F4.2A.

## 6. Métodos implementados vs stub

### AsaasProvider (implementados)

| Método | Comportamento |
|--------|---------------|
| `getName()` / `name` | `'asaas'` |
| `isEnabled()` | Lê `ASAAS_ENABLED` |
| `isConfigured()` | Enabled + `ASAAS_API_KEY` |
| `authenticate()` | API Key local; HTTP `GET /myAccount` se guards sandbox |
| `health()` | Idem authenticate (resumo + `ok`) |
| `validateWebhook()` | Stub — rejeita ou `ASAAS_STUB_NOT_IMPLEMENTED` |

### Stubs financeiros (não implementados)

| Método | Erro |
|--------|------|
| `createPixDeposit` | `ASAAS_PAYMENT_NOT_IMPLEMENTED` |
| `AsaasPaymentProvider.*` | Via stub acima |
| `createPixWithdraw` / `requestPixPayout` | `ASAAS_PAYOUT_NOT_IMPLEMENTED` |
| `getTransactionStatus` | `ASAAS_PAYMENT_*` / `ASAAS_PAYOUT_*` |
| Cobrança, QR, customer, billing, TED, transferência | **Não implementados** |
