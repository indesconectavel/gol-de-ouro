## 3. Flags e comportamento

| Flag | Default | Efeito |
|------|---------|--------|
| `CELCOIN_ENABLED` | `false` | Gate geral Celcoin |
| `CELCOIN_HTTP_ENABLED` | `false` | Gate chamadas HTTP reais |
| `ALLOW_CELCOIN_SANDBOX_AUTH` | (ausente) | Gate do script de teste (`=1` obrigatório) |

### `authenticate()` — matriz de decisão

| CELCOIN_ENABLED | CELCOIN_HTTP_ENABLED | Credenciais | Resultado |
|-----------------|----------------------|-------------|-----------|
| `false` | * | * | `CELCOIN_DISABLED` |
| `true` | `false` | * | `CELCOIN_STUB_NOT_IMPLEMENTED` |
| `true` | `true` | incompletas | `CELCOIN_NOT_CONFIGURED` |
| `true` | `true` | OK | `POST {authBaseUrl}/v5/token` → `access_token` |

Demais métodos (`createPixWithdraw`, etc.) **permanecem stub**.

---

## 4. Endpoint OAuth sandbox

| Item | Valor |
|------|-------|
| URL default | `https://sandbox.openfinance.celcoin.dev/v5/token` |
| Override | `CELCOIN_BASE_URL` (sem trailing slash) |
| Método | `POST` |
| Content-Type | `application/x-www-form-urlencoded` |
| Body | `client_id`, `grant_type=client_credentials`, `client_secret` |
| Resposta | `{ access_token, expires_in, token_type }` |

Documentação: [POST /v5/token](https://developers.celcoin.com.br/reference/post_v5-token)

- **Nunca** loga `client_secret` nem `access_token` completo

- Cache in-memory de token com skew de 30s antes do expiry
- Script de teste exige `ALLOW_CELCOIN_SANDBOX_AUTH=1` (padrão payout sandbox)
- mTLS **não** implementado nesta fatia (sandbox OAuth sem certificado)
