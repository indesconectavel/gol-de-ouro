# Diagnóstico

## O que funcionou

1. **Orquestração P1.6E.2** — JWT automático via fly ssh Windows-safe; auth admin HTTP 200.
2. **Janela operacional** — gates abertos/fechados automaticamente; rollback financeiro correto.
3. **Uma tentativa** — política respeitada; sem retry.

## Causa raiz provável

**Rejeição ou falha terminal no payout PIX pelo provedor Asaas** durante `createPixWithdrawCompat` → `POST /v3/transfers` (ou equivalente), **antes** da criação de `asaas_transfer_id`.

Indícios:

- HTTP 502 + `MP_TERMINAL_FAIL`
- `falha_payout` + `rollback` em ~200 ms
- `asaas_transfer_id` permanece `null`
- Padrão idêntico ao bloqueio **P1.5Y** (`invalid_action` / conta Asaas)

## Fatores contribuintes (não bloqueadores de auth)

| Fator | Impacto |
|-------|---------|
| `ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED=false` na janela | Webhook autorização não estava habilitado; irrelevante se transfer não foi criada |
| Saldo Asaas da conta | Possível (documentado P1.5Y.DIAG) — requer validação no painel Asaas |
| Saque em `falhou` | Impede reexecução com mesmo ID |

## O que NÃO foi a causa

- ❌ `invalid signature` JWT (eliminado P1.6E)
- ❌ Actor não-admin (eliminado P1.6E)
- ❌ `fly ssh` Windows (corrigido P1.6E.2)
- ❌ Gates não fechados (fechados com evidência)
