## Onde nasce `MP_TERMINAL_FAIL`

**Não é `throw`.** É **`return` explícito** no domain layer quando o provedor retorna `success: false`.

| Etapa | Arquivo | Linha | Mecanismo |
|-------|---------|-------|-----------|
| Provedor falha | `asaas-http-client.js` | 153–166 | `response.ok === false` → `return { success: false, error: 'ASAAS_API_REQUEST_FAILED', httpStatus, errorCode }` |
| Provider propaga | `AsaasProvider.js` | 228–234 | `if (!transferResult.success) return { ...transferResult }` |
| Domain classifica terminal | `payoutProviderPersistence.js` | 63–65 | `isPayoutTerminalFailure`: `payoutResult?.success === false` → `true` |
| Domain retorna código | `processPendingWithdrawals.js` | **1446–1451** | `return { success: false, code: 'MP_TERMINAL_FAIL', error: payoutResult?.error \|\| ... }` |
| Controller mapeia HTTP | `adminWithdrawController.js` | **291–298** | `MP_TERMINAL_FAIL: [502, 'Provedor rejeitou ou falhou no payout PIX']` |

# Parte 6 — Transfer Authorization (`ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED`)

## Estado na janela P1.6F

| Secret | Durante janela | Após fechamento |
|--------|----------------|-----------------|
| `ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED` | **false** | **false** |
| `ASAAS_TRANSFER_AUTH_WEBHOOK_TOKEN` | configurado (P1.6B.R2) | inalterado |

O script `openGates()` em `p16c-homolog-final.cjs` **não habilita** o gate de auth webhook — apenas gates PIX OUT financeiros. `closeGates()` força explicitamente `ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED=false`.

## Quando deve estar habilitado?

| Fase | Gate necessário? |
|------|------------------|
| **Criar** transferência (`POST /v3/transfers`) | ❌ Não — é chamada outbound com API Key |
| **Autorizar** transferência PENDING (~5 s depois) | ✅ Sim — endpoint `/webhooks/asaas/transfer-validation` deve responder `APPROVED` |

Fluxo oficial (P1.6 arquitetura):

```text
POST /v3/transfers → HTTP 200, transfer PENDING, asaas_transfer_id gravado
  ~5 s
Asaas → POST /webhooks/asaas/transfer-validation
  → APPROVED
Asaas executa PIX → webhook financeiro TRANSFER_DONE
```

```382:395:src/finance/webhooks/asaasTransferAuthorization.js
  if (!isAsaasTransferAuthWebhookEnabled()) {
    authLog('gate_disabled', { ... });
    return {
      httpStatus: 404,
      body: { status: 'REFUSED', refuseReason: 'Authorization endpoint disabled' },
      ...
    };
  }
```

## Impacto nesta falha

| Cenário | Impacto |
|---------|---------|
| Falha em `POST /v3/transfers` (P1.6F) | Gate auth **irrelevante** — transferência nunca criada |
| Próxima homologação com HTTP 200 | Gate auth **obrigatório** — sem ele, endpoint retorna 404 e transferência cancelada pelo Asaas |

**Conclusão:** `ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED=false` **não causou** o 502 de P1.6F, mas **impediria** conclusão mesmo se `POST /v3/transfers` retornasse 200.
