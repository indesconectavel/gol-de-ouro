## 3. Flags necessárias (pré-go-live PIX IN Asaas)

| Flag | Valor P1.3F / pré-go-live | Papel |
|------|---------------------------|-------|
| `ASAAS_ENABLED` | `true` | Ativa provider Asaas |
| `ASAAS_ENV` | `production` | Ambiente API produção |
| `ASAAS_PIX_IN_ENABLED` | `true` | Habilita PIX IN |
| `ASAAS_API_KEY` | chave `$aact_prod_…` (não vazia) | Autenticação API |
| `ASAAS_WEBHOOK_TOKEN` | token painel = Fly secret | Auth webhook |
| `ASAAS_WEBHOOK_ENABLED` | `true` | Webhook engine Asaas |
| `PAYMENT_WEBHOOK_ENGINE_ENABLED` | `true` | Payment webhook engine |
| `ASAAS_PRODUCTION_ENABLED` | **`false`** | Gate financeiro — **último passo go-live** |

**Rollback simples:** `ASAAS_PRODUCTION_ENABLED=false` → provider volta MP + webhook 403 + PIX IN Asaas bloqueado.

---

## 4. Fluxo de roteamento (pós-P1.3F)

```text
createPixDeposit()
  → isAsaasPixInHttpEnabled()              → sandbox HTTP (F4.4)
  → isAsaasPixInProductionHttpEnabled()    → produção HTTP (P1.3F) + gate ON
  → isAsaasPixInProductionHttpReady()      → ASAAS_PIX_IN_PRODUCTION_GATE_CLOSED
  → AsaasProvider stub                      → fallback legado interno

resolvePaymentProvider() [factory]
  → gate OFF + NODE_ENV=production         → mercadopago (inalterado)
  → gate ON + config completa              → asaas
```
