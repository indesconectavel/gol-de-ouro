## 9. Arquitetura

### Fluxos certificados

```text
Mercado Pago (IDs numéricos)
    ↓
RPC claim_and_credit_approved_pix_deposit
    ↓
Wallet + Ledger

Asaas (IDs pay_*)
    ↓
RPC → invalid_mercadopago_id
    ↓
Fallback JS (P1.4F) — somente neste caso
    ↓
Wallet + Ledger
```

| Componente | Arquivo |
|------------|---------|
| Orquestrador claim | `src/finance/deposit/claimApprovedPixDeposit.js` |
| Integração HTTP | `server-fly.js` → `claimAndCreditApprovedPixDeposit` |
| Condição fallback | `error === 'invalid_mercadopago_id'` + `pay_[a-z0-9]+` |

### RESSALVA ARQUITETURAL

O crédito Asaas em produção depende de **fallback JavaScript** enquanto a RPC permanece **MP-only** (`^\d+$`). Evolução futura recomendada: **RPC provider-agnostic** (Preferência B — P1.4F).

Adicionalmente, o crédito da cobrança homologada foi aplicado via **script controlado P1.4F** (`p14f-reprocess-asaas-deposit.cjs`) com gate fechado; o deploy v483 garante que **futuros webhooks** acionem o fallback automaticamente quando o gate for reaberto.
