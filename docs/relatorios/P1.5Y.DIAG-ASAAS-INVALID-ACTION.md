# DIAG-CONTA

A causa mais provável do `invalid_action` é **configuração da conta Asaas produção**: **autorização crítica habilitada** para transferências via API, sem mecanismo substituto (IP fixo, webhook de autenticação ou desabilitação via suporte). ### Capturada nos logs

| `transfer_id` | **null** (transferência não criada) |

### Não persistida no stdout

O client sanitiza erros em `sanitizeApiError()` e o evento `asaas_api_request_failed` registra apenas `httpStatus` e `errorCode` — **não** a `description` completa do body.

### Mensagem oficial Asaas para `invalid_action` (documentação)

Conforme [Security — Asaas](https://docs.asaas.com/docs/security):

```json
{
  "errors": [
    {
      "code": "invalid_action",
      "description": "Your account has critical authorization enabled. To perform this action, enter the confirmation code."
    }
  ]
}
```

Tradução PT ([FAQ Segurança](https://docs.asaas.com/docs/duvidas-frequentes-seguranca)):

> *"Sua conta possui autorização crítica habilitada. Para efetuar esta ação informe o código de confirmação."*

**Correlação:** o mesmo `errorCode` foi observado em sandbox F4.2G.2 quando transferências exigiam autorização crítica (SMS/APP/token).

## 5. Hipóteses (Fase 1 — Classificação)

| # | Hipótese | Prob. | Evidência |
|---|----------|-------|-----------|
| **1** | Conta sem permissão PIX OUT / transfers automáticos via API | **Alta** | `invalid_action` = autorização crítica na doc oficial; produção não envia `authToken` |
| **2** | API Key ambiente/escopo incorreto | Baixa | Prefixo `$aact_prod_`, base `api.asaas.com`, request aceito (400 de negócio, não 401) |
| **3** | KYC / conta produção incompleta | Média | Possível co-ocorrência; requer painel Asaas — não confirmado |
| **4** | Saldo / limite insuficiente | Média-baixa | Valor R$8 vs saldo painel ~R$3,02 citado em contexto operacional; **porém** Asaas documenta `invalid_action` para auth crítica, não saldo |
| **5** | Payload incompatível | **Baixa** | Campos batem OpenAPI; sandbox F4.2G criou transfer com mesmo shape |
| **6** | Tipo chave PIX incorreto | Baixa | `CPF` + 11 dígitos; validação interna passou |
| **7** | Endpoint incorreto | **Descartada** | `POST /v3/transfers` é o endpoint oficial PIX OUT |
| **8** | IP não autorizado (sem whitelist) | Baixa | Doc Asaas indica mensagem de IP inseguro — diferente de `invalid_action` |

**Classificação final:** hipótese **1** (conta / autorização crítica), com verificação pendente de **saldo (4)** e **KYC (3)** no painel.

## 9. Checklist do Painel Asaas (Fase 5)

Operador deve verificar manualmente:

- [ ] Conta **verificada** / documentação aprovada?
- [ ] **KYC** completo (PF/PJ conforme cadastro)?
- [ ] Saldo **disponível** ≥ R$ 8,00 (valor da tentativa R2)?
- [ ] API Key usada no Fly = mesma conta do painel (comparar ID/wallet)?
- [ ] **Transferências via API** habilitadas para a conta?
- [ ] **PIX OUT** habilitado (não apenas PIX IN/cobrança)?
- [ ] **Autorização crítica** — status atual?
  - [ ] Desabilitada com IP fixo (IP egress Fly.io)?
  - [ ] Ou webhook de autenticação configurado?
  - [ ] Ou ainda exige token SMS/APP a cada transferência?
- [ ] Limites diários / por transferência permitem R$ 8,00?
- [ ] Chave PIX destino (CPF `047***664`) válida e ativa no DICT?
- [ ] Perfil de negócio / segmento bloqueia saques?
- [ ] Necessário contato **gerente de conta** ou **suporte** para liberar automação?

### Ações Asaas documentadas para automação
