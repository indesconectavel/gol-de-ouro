## 2. Eventos Mapeados

Fonte: `asaas-webhook-validator.js` + docs Asaas webhooks.

| Evento Asaas | Status inferido | Status saque alvo |
|--------------|-----------------|-------------------|
| `TRANSFER_CREATED` | PENDING | aguardando_confirmacao |
| `TRANSFER_PENDING` | PENDING | aguardando_confirmacao |
| `TRANSFER_IN_BANK_PROCESSING` | BANK_PROCESSING | aguardando_confirmacao |
| `TRANSFER_DONE` | DONE | processado |
| `TRANSFER_FAILED` | FAILED | falhou |
| `TRANSFER_CANCELLED` | CANCELLED | falhou |
| `TRANSFER_BLOCKED` | BLOCKED | falhou |

**Identificadores de lookup (ordem):**

1. `saques.asaas_transfer_id` = `transfer.id`
2. `saques.payout_external_reference` = `transfer.externalReference`

**Payload esperado:**

```json
{
  "id": "evt_...",
  "event": "TRANSFER_DONE",
  "transfer": {
    "id": "trf_...",
    "status": "DONE",
    "externalReference": "goldeouro-...",
    "value": 10,
    "type": "PIX"
  }
}
```

## 6. Idempotência

| Condição | Comportamento |
|----------|---------------|
| Mesmo `asaas_transfer_status` + status saque já alinhado | `idempotent: true`, `replay: true` |
| Saque já terminal (`processado`/`falhou`) + mesmo target | Replay ignorado (só sync campos se status avançar) |
| `asaas_transfer_id` divergente do persistido | Erro 409 `TRANSFER_ID_MISMATCH` |
| Saque não encontrado | Erro 404 controlado `WITHDRAWAL_NOT_FOUND` |
