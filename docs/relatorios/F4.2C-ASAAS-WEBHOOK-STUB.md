## 5. Autenticação de webhook (documentação Asaas)

| Item | Valor confirmado |
|------|------------------|
| Mecanismo | **Token compartilhado** — **não** HMAC/Bearer OAuth |
| Header | `asaas-access-token` |
| Configuração | `authToken` no cadastro do webhook (painel ou API) |
| Referência | [About Webhooks](https://docs.asaas.com/docs/about-webhooks), [Create Webhook via API](https://docs.asaas.com/docs/create-new-webhook-via-api) |

### Lacunas documentais

| Lacuna | Status |
|--------|--------|
| Assinatura criptográfica (HMAC) | **Não documentada** — não implementada |
| Idempotência server-side oficial além do `id` do evento | **Parcial** — usamos `body.id` como event id; dedupe persistente fica para fatia posterior |
| Lista exhaustiva de todos os eventos Asaas | **Parcial** — mapeados pagamento + transferência; demais categorias (subscription, invoice, checkout) pendentes |

| Evento | Uso futuro |

Eventos fora dos conjuntos acima → `IGNORED_UNKNOWN_EVENT`.

## 7. Validações implementadas

| Check | Resultado |
|-------|-----------|
| Webhook desligado | `IGNORED_DISABLED` |
| Payload vazio / sem `event` / sem `id` | `INVALID_PAYLOAD` |
| Token inválido (strict) | `INVALID_SIGNATURE_OR_TOKEN` |
| Evento desconhecido | `IGNORED_UNKNOWN_EVENT` |
| Evento conhecido + estrutura OK | `VALID_SIMULATED_EVENT` |
| Efeito financeiro | **Sempre `false`** |
| PaymentProvider / PayoutProvider | **Não invocados** |
| HTTP real | **Não** |
| Banco / wallet / ledger | **Não alterados** |
