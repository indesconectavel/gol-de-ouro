**Data:** 2026-06-27  
**Fatia:** classificação de eventos webhook Asaas Sandbox — **sem efeito financeiro**  
**Produção:** inalterada — **nenhuma rota pública habilitada**

---

## 1. Objetivo

Validar recepção (simulada/local) e classificação segura de eventos webhook do Asaas Sandbox, confirmando que:

- o handler classifica payloads conhecidos corretamente;
- token/header `asaas-access-token` são validados em modo strict;
- payloads inválidos e eventos desconhecidos são rejeitados/ignorados;
- **nenhum** saldo, ledger, wallet ou fluxo financeiro do Gol de Ouro é alterado;
- toda resposta inclui `financialEffect: false`.

Base preparatória para homologação futura de PIX IN/OUT com segurança.

## 3. Estratégia de validação

### Prioridade 1 — Evento real via URL pública/tunnel

**Não aplicável nesta fatia.** Não há tunnel local nem URL pública configurada para receber POST do Asaas Sandbox.

**Decisão de segurança:** nenhuma rota HTTP pública foi montada ou habilitada em produção. Recepção real ficará para fatia posterior, com gate explícito e ambiente controlado.

### Prioridade 2 — Payload simulado + payload real opcional

- Cenários simulados cobrem todos os eventos prioritários de pagamento e transferência.
- Payload real opcional via arquivo local:

```powershell
$env:ASAAS_WEBHOOK_PAYLOAD_FILE = "tmp/asaas-webhook-payload.json"
node scripts/test-asaas-webhook-live.mjs
```

Se o arquivo não existir, o script informa `realPayload: not provided` e usa payload realista baseado na cobrança F4.2E (`pay_xchnwrdb321lqggq`, evento `PAYMENT_RECEIVED`).

## 4. Feature flags

| Flag | Default | Efeito |
|------|---------|--------|
| `ASAAS_WEBHOOK_ENABLED` | `false` | Gate principal — `IGNORED_DISABLED` |
| `ASAAS_WEBHOOK_STRICT_MODE` | `true` | Exige `ASAAS_WEBHOOK_TOKEN` + header válido |
| `ASAAS_WEBHOOK_TOKEN` | (vazio) | Token esperado no header |
| `ASAAS_WEBHOOK_LIVE_TEST` | `0` | Modo live sandbox de classificação |
| `ALLOW_ASAAS_SANDBOX_WEBHOOK` | `0` | Permite validação live sandbox |

Gate combinado (`isAsaasWebhookLiveSandboxActive`):

```text
ASAAS_WEBHOOK_ENABLED=true
ASAAS_WEBHOOK_LIVE_TEST=1
ALLOW_ASAAS_SANDBOX_WEBHOOK=1
ASAAS_ENV=sandbox
```

Com flags desligadas → nenhum evento processado (`IGNORED_DISABLED`).  
Com flags ligadas → apenas classificação; decisões `VALID_SANDBOX_*` sem efeito financeiro.

## 6. Decisões do handler

| Decisão | Quando |
|---------|--------|
| `IGNORED_DISABLED` | `ASAAS_WEBHOOK_ENABLED=false` |
| `INVALID_PAYLOAD` | body vazio, sem `event` ou sem `id` |
| `INVALID_SIGNATURE_OR_TOKEN` | token ausente/incorreto (strict) |
| `IGNORED_UNKNOWN_EVENT` | evento fora do mapa conhecido |
| `VALID_SIMULATED_EVENT` | F4.2C — webhook habilitado sem live test |
| `VALID_SANDBOX_EVENT` | Live sandbox — categoria genérica |
| `VALID_SANDBOX_PAYMENT_EVENT` | Live sandbox — evento de pagamento |
| `VALID_SANDBOX_TRANSFER_EVENT` | Live sandbox — evento de transferência |

**Todas** as respostas incluem `financialEffect: false`.

Fases do handler:

- `webhook_stub_v1` — F4.2C (simulado)
- `webhook_live_sandbox_v1` — F4.2F (classificação live sandbox)
