# Diffs resumidos

## Correção 1 — `p16c-final-precheck.mjs`

**Removido:** expectativa HTTP 403 `asaas_webhook_blocked` / `ASAAS_WEBHOOK_NOT_PRODUCTION_GATED`

**Adicionado:**

```javascript
function isWebhookIngressHealthy(status, body = {}) {
  if (status === 403 && body?.error === 'asaas_webhook_blocked') return false;
  if (status === 404 || status >= 500) return false;
  if (status === 200 || status === 401) return true;
  return false;
}
```

- Check renomeado: `asaas_webhook_ingress_validated` (antes `asaas_webhook_production_gated`)
- Auth webhook em repouso: 404 `asaas_transfer_auth_webhook_disabled` = **OK**
- Verdicts: `ARCHITECTURE_READY` (exit 0) vs `READY_TO_EXECUTE` (saque pendente + chave correta)

# Justificativa

P1.6W desacoplou o **ingresso** do webhook financeiro de `ASAAS_PRODUCTION_ENABLED`. O precheck legado tratava HTTP 403 como sinal de segurança — após P1.6W isso indica **regressão**.

P1.6F falhou parcialmente porque `ASAAS_TRANSFER_AUTH_WEBHOOK_ENABLED` permaneceu false na janela. `openGates()` agora habilita auth webhook junto com PIX OUT.

Chave PIX de homologação atualizada para conta operacional real (`indesconectavel@gmail.com`).
