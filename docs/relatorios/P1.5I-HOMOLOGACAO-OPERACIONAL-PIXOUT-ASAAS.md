## 6. Evidências Técnicas

### Implementação ledger (P1.5I)

Arquivo: `src/finance/webhooks/processAsaasTransferWebhook.js`

| Função | Responsabilidade |
|--------|------------------|
| `findExistingTerminalLedger` | Idempotência MP-style (`payout_confirmado` / `falha_payout`) |
| `applyTerminalLedgerReconciliation` | DONE → ledger confirmado; FAIL → ledger + `rollbackWithdraw` |
| `parseSaqueAmounts` | amount / fee / net_amount do saque |

Comportamento:

- **DONE:** `createLedgerEntry({ tipo: 'payout_confirmado' })` — wallet **não** alterada (débito já ocorreu na solicitação).
- **FAILED / CANCELLED / BLOCKED:** `createLedgerEntry({ tipo: 'falha_payout' })` + `rollbackWithdraw()` — restaura saldo e registra `rollback` no ledger.
- **Replay:** `findExistingTerminalLedger` retorna existente → `ledgerIdempotent: true`, sem efeito financeiro duplicado.

Flag removida: `ledgerPending` (stub P1.5H).
