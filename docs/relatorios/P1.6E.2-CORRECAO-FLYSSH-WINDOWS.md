# Resumo Executivo

Corrigida a invocação `fly ssh console` no homologador para Windows. A causa raiz (P1.6E.1) era `shell: true` no `spawnSync`, que fazia o cmd reinterpretar `-C printenv JWT_SECRET` e gerar falso negativo DNS (`host unavailable at JWT_SECRET`).

Após P1.6E.2:

- `JWT_SECRET` obtido automaticamente via `fly ssh` em **~3,3 s**;
- `auth-preflight` concluiu com **`probeOk: true`** e HTTP **200**;
- exit code **1** com `Identificador inválido` tratado como ruído Windows quando o parser confirma secret válido;
- **nenhuma** alteração em backend, JWT de produção, Payment Engine, webhooks ou operações financeiras.

---

# Problema

| Antes (P1.6E) | Sintoma |
|---------------|---------|
| `spawnSync(..., { shell: true })` no Windows | cmd divide `printenv JWT_SECRET` → fly interpreta `JWT_SECRET` como hostname |
| Abort por `exit=1` | Mesmo com stdout válido, homologador falhava |
| `auth-preflight` 2026-06-30T06:50Z | `fly ssh exit=1` em ~6 s, sem secret parseável |

# Correção implementada

## 1. `shell: false`

`spawnSync('fly', flyArgs, { shell: false, ... })` — argumentos passados diretamente ao `fly.exe`, sem reinterpretação do cmd.

## 2. Argumentos separados

```javascript
['ssh', 'console', '-q', '-a', app, '-C', 'printenv JWT_SECRET']
```

Cada token individual; sem string única montada manualmente.

## 3. Parser em streams separados

- `parseFlyJwtSecretFromStreams(stdout, stderr)` — prioriza **stdout**;
- rejeita linhas críticas (`host unavailable`, DNS, `permission denied`, etc.);
- preserva regex/comprimento da P1.6E.

## 4. Tratamento de exit code

| Condição | Resultado |
|----------|-----------|
| Secret válido + stderr só teardown Windows | **SUCCESS** + warning |
| Secret válido + exit 0 | **SUCCESS** |
| Sem secret | **FAIL** |
| Erro crítico (DNS/host/timeout) | **FAIL** |
| Secret + exit≠0 + stderr desconhecido | **FAIL** |
| `signal` (timeout) | **FAIL** |

Teardown Windows reconhecido: `Identificador inválido`, `Invalid identifier`, `ssh shell: Process exited with status`, linhas `Connecting to …`.

## 5. Logs (sem secrets)

```text
Executando fly ssh (Windows-safe)…
stdout recebido.
stderr analisado.
parser OK.
JWT encontrado (valor omitido).
Warning: Fly SSH encerrou com erro conhecido do Windows. JWT obtido com sucesso.
Erro conhecido do Windows ignorado.
Prosseguindo.
```
