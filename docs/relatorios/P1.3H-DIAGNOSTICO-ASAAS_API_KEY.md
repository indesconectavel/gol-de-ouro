## Conclusão

# Secret realmente vazio no runtime (injeção Fly)

A variável `ASAAS_API_KEY` **existe** no ambiente do processo Node como **string vazia** (`""`), não como `undefined`. O código lê o nome **correto** (`process.env.ASAAS_API_KEY`). **Não há** evidência de leitura de variável alternativa nem de sobrescrita por `.env`, `fly.toml` ou lógica da aplicação.

A causa mais provável é **gravação histórica do secret com valor vazio** no Fly (`fly secrets set ASAAS_API_KEY=` com expansão de shell vazia ou comando sem valor), **anterior** à sessão P1.3G. O comando P1.3G **não** tocou em `ASAAS_API_KEY`.

**Não é** um bug de runtime lendo variável incorreta.
