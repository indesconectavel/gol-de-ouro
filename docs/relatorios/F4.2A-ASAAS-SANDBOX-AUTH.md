# F4.2A — Asaas Sandbox HTTP Client (Auth / Conectividade)

**Data:** 2026-06-27  
**Fatia:** configuração + cliente HTTP isolado + script de teste — **sem PIX, cobrança, cliente ou saque**  
**Produção:** inalterada (`ASAAS_ENABLED=false`, `ALLOW_ASAAS_SANDBOX_AUTH=0` por default)

---

## 1. Objetivo

Implementar a primeira etapa isolada da integração Asaas Sandbox na Payment Engine:

- configuração via variáveis de ambiente;
- cliente HTTP mínimo com guards de segurança;
- script local de autenticação/conectividade (`GET /myAccount`);
- relatório desta fatia.

Nenhum fluxo de wallet, ledger, gameplay, Mercado Pago ou Celcoin é tocado nesta etapa.

## 3. Variáveis necessárias

```env
ASAAS_ENABLED=false
ASAAS_ENV=sandbox
ASAAS_API_KEY=
ASAAS_BASE_URL=https://sandbox.asaas.com/api/v3
ALLOW_ASAAS_SANDBOX_AUTH=0
```

### Matriz de guards

| ASAAS_ENABLED | ALLOW_ASAAS_SANDBOX_AUTH | ASAAS_ENV | ASAAS_API_KEY | HTTP real |
|---------------|--------------------------|-----------|---------------|-----------|
| `false` | * | * | * | **bloqueado** |
| `true` | `0` | * | * | **bloqueado** |
| `true` | `1` | ≠ `sandbox` | * | **bloqueado** |
| `true` | `1` | `sandbox` | ausente | **bloqueado** |
| `true` | `1` | `sandbox` | OK | `GET /myAccount` |

## 5. Resultado esperado

### Default (guards desligados)

```powershell
node scripts/test-asaas-auth.mjs
```

Saída esperada: `result: "SKIP"`, `httpCalled: false`, exit code `0`.

### Sandbox autorizado (com chave válida)

```powershell
$env:ALLOW_ASAAS_SANDBOX_AUTH = '1'
$env:ASAAS_ENABLED = 'true'
$env:ASAAS_ENV = 'sandbox'
$env:ASAAS_API_KEY = '$aact_hmlg_...'
node scripts/test-asaas-auth.mjs
```

Saída esperada: `result: "PASS"`, dados resumidos da conta (`id`, `name`, `email`), exit code `0`.

Falha de credencial/rede: `result: "FAIL"`, exit code `2`.

---

## 6. Segurança

- A chave `ASAAS_API_KEY` **nunca** aparece inteira em logs ou stdout.
- Preview mascarado: `$aact_hmlg_abc...xyz` ou `***masked***`.
- HTTP real exige três gates: `ASAAS_ENABLED=true`, `ALLOW_ASAAS_SANDBOX_AUTH=1`, `ASAAS_ENV=sandbox`.
- Apenas endpoint de leitura nesta fatia.

---

## 7. Riscos

| Risco | Mitigação |
|-------|-----------|
| Chamada acidental em produção | `ASAAS_ENV` deve ser `sandbox`; guards default desligados |
| Exposição de API key em logs | Mascaramento centralizado em `maskApiKeyPreview()` |
| Escopo creep (PIX/cobrança) | Cliente expõe somente `fetchMyAccount`; sem rotas/wallet alteradas |
| URL base incorreta | Override via `ASAAS_BASE_URL`; documentação oficial usa também `api-sandbox.asaas.com` |

---
