## 2. Etapa 1 — Cadeia de código

### 2.1 Fluxo PIX IN

```text
POST /api/payments/pix/criar          [server-fly.js L3045+]
  → authenticateToken
  → isPixDepositConfigured()
  → createPixDepositCompat            [createPixDepositCompat.js]
       → createPixDeposit             [createPixDeposit.js]
            → resolvePaymentProvider() [FinanceProviderFactory.js]
            → provider.createPixDeposit()
                 [mercadopago] MercadoPagoPaymentProvider — HTTP MP prod ✅
                 [asaas]       AsaasPaymentProvider
                      → isAsaasPixInHttpEnabled()?
                           false (produção) → AsaasProvider.createPixDeposit() → **STUB** ❌
                           true  (sandbox)  → createPixChargeSandbox() → API sandbox ✅
```

### 2.2 Pontos de integração

| Componente | Arquivo | Papel |
|------------|---------|-------|
| Rota HTTP | `server-fly.js` | `POST /api/payments/pix/criar` |
| Compat | `src/finance/compat/createPixDepositCompat.js` | Ponte monólito → engine; `financialEffect: false` na criação |
| Engine | `src/finance/deposit/createPixDeposit.js` | Entry point provider-agnostic |
| Factory | `src/finance/factory/FinanceProviderFactory.js` | `resolvePaymentProvider()` + gates F4.3C |
| Asaas adapter | `src/finance/providers/asaas/AsaasPaymentProvider.js` | PIX IN sandbox HTTP; **stub em produção** |
| Gate PSP | `src/finance/config/primary-psp.js` | Prod sem gate → provider efetivo `mercadopago` |

### 2.3 Condição `isAsaasPaymentProviderResolvable()`

Asaas só é provider de PIX IN em produção quando **simultaneamente**:

| Requisito | Função / flag |
|-----------|---------------|
| Gate produção | `ASAAS_PRODUCTION_ENABLED=true` |
| Runtime | `NODE_ENV=production` |
| Asaas ativo | `ASAAS_ENABLED=true` |
| Configurado | `ASAAS_API_KEY` presente |
| Ambiente | `ASAAS_ENV≠sandbox` |

### 2.4 Achado crítico — HTTP PIX IN produção

`AsaasPaymentProvider.createPixDeposit()` só chama API HTTP quando `isAsaasPixInHttpEnabled()` — gate **exclusivo sandbox**:

```text
isAsaasPixInHttpEnabled()
  → isAsaasHttpEnabled()        // exige ASAAS_ENV=sandbox
  → isAsaasPixInEnabled()
  → isAsaasSandboxPixInAllowed() // ALLOW_ASAAS_SANDBOX_PIX_IN=1
```

Em produção, o fluxo cai em `AsaasProvider.createPixDeposit()` → **`_paymentStub`** (`create_pix_deposit_blocked_stub`).

**Conclusão:** mesmo com `ASAAS_PRODUCTION_ENABLED=true`, o backend **não cria cobrança PIX IN real via API Asaas produção** hoje. Confirmado em P1.1A; revalidado nesta auditoria.

### 6.2 Conclusão

# FAIL

A auditoria P1.3 **confirma** que o gate controlado está **correto** e que depósitos em produção seguem pelo **Mercado Pago**. Porém o objetivo **PIX IN real Asaas produção** **não pode ser validado nem executado** nesta sessão por:

1. **Código:** `AsaasPaymentProvider.createPixDeposit` não possui caminho HTTP para produção — apenas sandbox ou stub (P1.1A D6).
2. **Infra:** `ASAAS_API_KEY` vazia no runtime Fly — API Asaas produção inacessível (P1.2).
3. **Flags:** `ASAAS_ENABLED`, `ASAAS_ENV`, `ASAAS_PIX_IN_ENABLED` ausentes no runtime — Asaas não configurado para PIX IN mesmo com gate futuro.
4. **Gate:** `ASAAS_PRODUCTION_ENABLED=false` — intencional; impede resolver Asaas (comportamento esperado).

**Interpretação:** P1.3 documenta o **estado seguro atual** e os **bloqueadores** antes de qualquer teste PIX IN real. O próximo passo de engenharia é implementar HTTP PIX IN produção; o próximo passo operacional é corrigir `ASAAS_API_KEY` no Fly.
